    JavaScript Questions

    JavaScript is an interpreted, single-threaded language which is helps to make any website interactive. Interpreted means the source code is converted into bytecode and executed at runtime. Initially, it was only interpreted, but now for better performance, JS engines use different ways to perform Just-in-Time Compilation (JIT). Single-threaded refers that it can only run one operation at a time.

    It has some abilities which makes it unique from all other languages. In JS, one object can inherit features from some other objects. It is also known as dynamic weakly typed language. Because, variables are not declared with data types as int, string etc. and the actual data types are unknown until runtime.

    Despite of being a single-threaded language, JavaScript provides an illusion of multi-threaded system and event loop is the reason behind it. The call stack in JS, keeps track of all the functions that are to be executed. When a new function is received, it is pushed into the stack and after completion of the function, it is popped out of the stack. The event queue in JS, sends new functions to the call stack for processing. The event loop is a constantly running process that monitors both the queue and the stack. It constantly checks whether the stack is empty or not. If it is empty, new functions are included from the event queue. If it is not, then the current function is processed. When an async function is called, it is sent to a browser API and the API starts its own single-threaded operation and at the same time, other functions are executed hence behaving like a multi-threaded system.
